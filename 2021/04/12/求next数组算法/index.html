这篇学求next数组算法

话不多说，直接上代码

void GetNext(SString t,int next[]) {  
    int j, k;
    j=0;     
    k=-1;    
    next[0]=-1;
    while (j<t.length-1) 
    //因为next数组的赋值都是在j++之后，所以判断条件是j<t.length-1 
    {
        if (k==-1 || t.data[j]==t.data[k]) {
            j++; //推进到下一位
            k++;//推进到下一位
            next[j]=k;    /*更新next数组*/
        }
        else
            k=next[k];  
             //next[k]代表的是下标k之前的字符的前缀后缀相同最大值
             //所以next[k]也同时代表了字符不匹配时k回溯的位置（有点难理解，可以参考KMP算法的图解，差不多的原理）
             //回溯到最长相同前缀的后一位（下面会讲为什么）
             
                }
}

虽然代码很短，但是理解起来十分费劲，不得不佩服前人的智慧，tql。

让我们来说说为什么k=next[k]之后k的位置是最长相同前缀的后一位

  • next[k]代表的是最长相同前缀的长度,所以最长相同前缀最后一个字符的下标对应就是next[k]-1，因此next[k]就是下一位。
  • 光这么说，还是很难理解，我们上一个例子。

例子

  • 演算过程:

  • 初始:j=0; k=-1; next[0]=-1;

  • 1）j=1; k=0; next[1]=0;

  • 2）t.data[1] t.data[0] 不等 k=next[0]=-1;

  • 3）j=2; k=0; next[2]=0;

  • 4）t.data[2] t.data[0] 不等 k=next[0]=-1;

  • 5）j=3; k=0; next[3]=0;

  • 6）t.data[3] = =t.data[0] 等 

    j=4; k=1; next[4]=1;

  • 7）t.data[4] = =t.data[1] 等 

    j=5; k=2; next[5]=2;

    这算法确实难理解,再次感叹先人智慧.
